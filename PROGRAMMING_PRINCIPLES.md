# Принципи програмування в проєкті

1. MVC (Model-View-Controller)
Архітектура проєкту розділена на три компоненти: Модель, Представлення та Контролер.

Model (Модель): Визначення структури даних та взаємодія з базою даних.
Файл: models.py
Приклад: Клас User (рядки 8-21), Car (рядки 34-51).

View (Представлення): HTML-шаблони, що відповідають за інтерфейс користувача.
Директорія: templates/
Приклад: templates/index.html

Controller (Контролер): Обробка HTTP-запитів та керування потоком даних.
Файл: app.py
Приклад: Функція index() (рядки 44-57).

2. SRP (Single Responsibility Principle)
Кожен модуль або клас відповідає лише за одну частину функціоналу системи.

Сервіси: Логіка винесена в окремі сервісні файли.
Файли:
- services/auth_service.py: Автентифікація та реєстрація.
- services/booking_service.py: Логіка бронювання.
- services/car_service.py: Операції з автомобілями.

Контролери: app.py відповідає лише за маршрутизацію.

3. DRY (Don't Repeat Yourself)
Уникнення дублювання коду.

Enums: Використання перелічень замість рядків.
Файл: enums.py
Приклад: UserRole, BookingStatus. Використовуються в models.py (рядки 14, 48, 63).

Шаблони: Використання наслідування шаблонів Jinja2.
Файл: templates/base.html

Декоратори: Повторювана логіка перевірки прав доступу.
Файл: app.py
Приклад: Декоратор role_required (рядки 33-41).

4. Service Layer Pattern
Бізнес-логіка відокремлена від контролерів.

Файли: Директорія services/
Приклад: Виклик process_booking у app.py (рядок 101).

5. KISS (Keep It Simple, Stupid)
Код написаний просто, без зайвої складності.

Приклад: Функція validate_phone в services/booking_service.py (рядки 5-6).

6. Dependency Injection
Явна ініціалізація залежностей.

Файл: app.py
Приклад: db.init_app(app) (рядок 24).
